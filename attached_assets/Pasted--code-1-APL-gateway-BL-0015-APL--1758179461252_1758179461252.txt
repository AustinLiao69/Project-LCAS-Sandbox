1. APL gateway不能直接調用BL層嗎？
技術上可以，但不符合架構設計。根據0015文件的架構定義：
•	APL層定義：「是 PL 與 BL 之間的橋樑，由RESTful API 端點與格式規格組成，提供統一的介面規範」
•	架構原則：APP端必須透過APL的RESTful API與BL溝通，而LINE OA可以直接呼叫BL
所以APL Gateway應該是純路由轉發，不應該直接調用BL函數。

2. Express HTTP路由層這個是0015文件的技術架構有定義的嗎？
沒有明確定義Express HTTP路由層。0015文件中的架構是這樣的：
APP端 → APL (RESTful API) → BL → DL
LINE OA → BL → DL (直接調用)
0015文件只定義了APL是「RESTful API 端點與格式規格組成」，但沒有說明這些端點的實作方式。Express HTTP路由層是實作這些RESTful API端點的技術方案。

=>如果要這樣的話，那必須要寫DCN以及更新0015文件，還有其下階文件(SRS, LLD等)，請你提出一個相應的計劃。
