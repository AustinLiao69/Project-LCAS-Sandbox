## 3. 使用情境清單（User Scenarios）

### ▶ Scenario 001：精準控制者的專案投資管理

- **角色**：精準控制者（高動機+高自律）
- **目標**：建立並管理2025年投資專案，追蹤每筆投資的ROI和風險分配
- **觸發條件／情境背景**：年初制定投資計畫，需要精細化管理多個投資標的
- **使用裝置／平台**：主要使用Mobile APP，搭配LINE OA快速記帳
- **操作流程簡述**：
  1. APP中建立「2025投資專案」專案帳本
  2. 設定總預算$500,000，分配至股票、基金、加密貨幣三類
  3. 每日透過LINE OA快速記錄股息收入：「收入 股息 1500 台積電配息」
  4. 週末使用APP詳細分析投資績效，產生專業報表
  5. 月底匯出投資報表給理財顧問討論
- **期望結果**：完整掌握投資組合狀況，ROI追蹤準確度>95%，決策支援完整
- **潛在阻礙／風險點**：
  - 跨平台資料同步延遲影響即時分析
  - 複雜報表產生時間過長
  - 投資科目分類不夠細緻
- **與其他模組關聯**：F031專案帳本建立、F043標準報表產出、F045報表匯出功能

### ▶ Scenario 002：紀錄習慣者的美感記帳體驗

- **角色**：紀錄習慣者（低動機+高自律）
- **目標**：維持優雅的每日記帳習慣，享受記錄過程的美感與儀式感
- **觸發條件／情境背景**：每天晚上9點的固定記帳時間，在咖啡廳或家中安靜環境
- **使用裝置／平台**：以APP為主，重視視覺體驗和操作流暢度
- **操作流程簡述**：
  1. 收到晚上9點的溫和提醒：「🌙 該為今天劃下美好句號了」
  2. 開啟APP，欣賞當日主題配色（櫻花粉或湖水藍）
  3. 優雅地輸入一天的支出：午餐$280、咖啡$120、書籍$350
  4. 選擇相應的美感科目圖示，為每筆記錄加上個人標籤
  5. 查看當月收支圖表，享受數據視覺化的美感
  6. 系統自動備份，無需擔心資料安全
- **期望結果**：記帳成為生活儀式，持續記錄率>90%，體驗滿意度極高
- **潛在阻礙／風險點**：
  - APP介面設計不夠美觀影響使用意願
  - 記帳流程過於複雜破壞儀式感
  - 提醒訊息過於商業化缺乏溫度
- **與其他模組關聯**：F007 APP記帳功能、F049排程提醒設定、F051定期自動備份

### ▶ Scenario 003：轉型挑戰者的習慣養成之路

- **角色**：轉型挑戰者（高動機+低自律）
- **目標**：從月光族轉變為有預算觀念的理財新手，建立穩定記帳習慣
- **觸發條件／情境背景**：剛換工作加薪，決心改善財務狀況，但常常忘記記帳
- **使用裝置／平台**：LINE OA為主要工具，APP作為進階分析使用
- **操作流程簡述**：
  1. 在APP中設定月預算$35,000，分配生活費、娛樂費、儲蓄
  2. 設定多重提醒：餐後提醒、晚間總結、週末檢討
  3. 用餐後收到LINE提醒：「🍽️ 別忘了記錄剛剛的美食！」
  4. 回傳「支出 午餐 180」，系統立即回饋：「✅ 已記錄！本月餐飲預算還剩$2,300」
  5. 預算警示：「⚠️ 娛樂費已用80%，建議本週控制支出」  
  6. 邀請室友加入共享帳本，互相督促記帳
  7. 週末查看APP報表：「🎉 本週記帳率85%，比上週進步了！」
- **期望結果**：3個月內建立穩定記帳習慣，預算執行率>75%，儲蓄目標達成
- **潛在阻礙／風險點**：
  - 提醒過於頻繁造成反感
  - 預算警示過於嚴厲打擊信心
  - 缺乏成就感導致中途放棄
- **與其他模組關聯**：F037預算設定建立、F038預算追蹤監控、F040共享帳本建立、F049排程提醒設定

### ▶ Scenario 004：潛在覺醒者的財務啟蒙之旅

- **角色**：潛在覺醒者（低動機+低自律）
- **目標**：在不知不覺中開始關注個人財務，逐步建立金錢觀念
- **觸發條件／情境背景**：朋友推薦加入LCAS LINE官方帳號，抱著試試看的心態
- **使用裝置／平台**：完全依賴LINE OA，簡單直覺的互動
- **操作流程簡述**：
  1. 朋友分享LINE官方帳號，好奇心驅使下加入
  2. 收到溫暖歡迎訊息：「👋 歡迎來到LCAS！無需設定，現在就能開始記錄」
  3. 第一次嘗試：「支出 午餐 120」
  4. 系統驚喜回應：「🎉 恭喜完成第一筆記錄！今天花了$120在美食上」
  5. 幾天後，系統溫和提醒：「💡 這週記錄了5筆，總共$850，想看看花在哪了嗎？」
  6. 點擊查看，驚訝發現：「原來我一週在飲料上花了$200！」
  7. 系統智慧引導：「要不要試試設個小目標？比如飲料費控制在$150？」
  8. 收到可愛的視覺化圖表：「📊 你的消費大餅圖出爐啦！」
- **期望結果**：在無壓力環境下逐步建立記帳習慣，2個月內自發記錄率>50%
- **潛在阻礙／風險點**：
  - 引導過於主動造成抗拒
  - 功能介紹過於複雜嚇跑用戶
  - 缺乏持續的動機維持
- **與其他模組關聯**：F001使用者註冊、F009 APP記錄查詢、F043標準報表產出

### ▶ Scenario 005：跨模式協作情境 - 家庭財務管理

- **角色**：精準控制者（父親）+ 轉型挑戰者（母親）+ 潛在覺醒者（子女）
- **目標**：建立家庭共同財務管理系統，培養全家理財觀念
- **觸發條件／情境背景**：搬新家後希望全家一起管理家庭開支和儲蓄目標
- **使用裝置／平台**：跨平台協作，依個人偏好選擇工具
- **操作流程簡述**：
  1. 父親（精準控制者）在APP建立「張家家庭帳本」共享帳本
  2. 設定家庭預算：生活費$50,000、教育費$20,000、娛樂費$15,000
  3. 邀請母親和子女加入，分配不同權限等級
  4. 母親（轉型挑戰者）主要透過LINE記錄採買：「支出 菜錢 450 全聯採購」
  5. 子女（潛在覺醒者）記錄零用錢使用：「支出 文具 80」
  6. 父親週末產出家庭財務報表，全家一起檢討
  7. 系統智慧分析：「本月教育支出比預期少30%，可考慮增加課外活動」
  8. 達成儲蓄目標時，全家收到慶祝訊息：「🎉 恭喜達成旅遊基金目標！」
- **期望結果**：家庭財務透明化，預算達成率>80%，全家理財觀念提升
- **潛在阻礙／風險點**：
  - 不同模式用戶的需求衝突
  - 權限管理過於複雜
  - 隱私與透明度的平衡
- **與其他模組關聯**：F040共享帳本建立、F041多人協作權限、F042即時協作同步

### ▶ Scenario 006：模式發現與切換情境

- **角色**：紀錄習慣者 → 精準控制者（模式升級）
- **目標**：隨著理財知識增長，從簡單記錄進階為深度分析
- **觸發條件／情境背景**：使用3個月後，開始對投資理財產生興趣
- **使用裝置／平台**：從LINE OA為主轉為APP深度使用
- **操作流程簡述**：
  1. 系統發現用戶記帳行為變化：開始詢問投資相關問題
  2. 智慧推薦：「🤖 發現您對投資有興趣，要不要試試專案帳本功能？」
  3. 引導建立第一個投資專案帳本
  4. 解鎖進階功能：預算管理、報表分析、資產配置
  5. 系統個人化調整：介面變得更專業，功能更豐富
  6. 舊資料自動遷移至新模式，無縫切換體驗
  7. 提供模式切換教學：「📚 歡迎進入精準控制者模式！」
- **期望結果**：平滑的功能升級體驗，用戶能力與工具複雜度匹配
- **潛在阻礙／風險點**：
  - 功能突然變複雜造成使用困擾
  - 模式識別不準確推薦錯誤功能
  - 升級過程資料遺失風險
- **與其他模組關聯**：模式切換機制、F031專案帳本建立、使用者設定管理

### ▶ Scenario 007：離線使用情境

- **角色**：轉型挑戰者
- **目標**：在網路不穩定環境下仍能正常記帳
- **觸發條件／情境背景**：出國旅遊期間，網路連線不穩定
- **使用裝置／平台**：Mobile APP離線功能
- **操作流程簡述**：
  1. 出國前APP自動提醒：「🌍 偵測到您可能要出國，已開啟離線模式」
  2. 在飛機上記錄：「支出 機場餐廳 280」，APP提示「✈️ 離線記錄已儲存」
  3. 到達目的地後，記錄住宿和交通費用
  4. APP顯示：「📱 已暫存5筆記錄，連線後自動同步」
  5. 連上飯店WiFi後，系統自動同步所有離線記錄
  6. 同步完成通知：「🔄 歡迎回來！5筆記錄已同步，旅遊預算剩餘$15,000」
- **期望結果**：離線記錄無遺失，同步準確率100%，用戶體驗不受網路影響
- **潛在阻礙／風險點**：
  - 離線記錄儲存失敗
  - 同步時發生資料衝突
  - 離線模式功能受限
- **與其他模組關聯**：離線暫存機制、F046數據同步檢查

### ▶ Scenario 008：錯誤處理與恢復情境

- **角色**：精準控制者
- **目標**：快速發現並修正記帳錯誤，維持資料準確性
- **觸發條件／情境背景**：發現昨天記錯一筆大額支出金額
- **使用裝置／平台**：Mobile APP
- **操作流程簡述**：
  1. 查看報表時發現異常：「昨天餐費$2,800 seems too high」
  2. 使用搜尋功能快速定位錯誤記錄
  3. 點擊編輯，發現是$280誤輸入為$2,800
  4. 修正金額後，系統提醒：「⚠️ 這會影響昨日統計和預算計算」
  5. 確認修正後，相關報表自動更新
  6. 系統學習錯誤模式，下次輸入類似金額時會提醒確認
  7. 修正歷史記錄在協作帳本中透明顯示
- **期望結果**：錯誤能被快速發現和修正，不影響長期數據分析
- **潛在阻礙／風險點**：
  - 編輯權限不足無法修正
  - 修正操作過於複雜
  - 影響他人的協作記錄
- **與其他模組關聯**：F009 APP記錄查詢、記錄編輯功能、協作權限管理
