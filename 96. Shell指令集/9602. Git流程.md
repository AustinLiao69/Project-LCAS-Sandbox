
# 9602. Git流程

## 00. Branch push到main流程

需要替換 `your-branch-name` 為你實際的branch名稱。

### 確認現在在哪個branch
```bash
git branch
```

### 檢查當前狀態
```bash
git status
```

### 合併流程
確認狀態後，按照以下順序執行：

1. **切換到main branch：**
   ```bash
   git checkout main
   ```

2. **拉取最新的main branch變更：**
   ```bash
   git pull origin main
   ```

3. **合併你的branch到main（請替換成你實際的branch名稱）：**
   ```bash
   git merge Sandbox
   ```

4. **推送到remote main branch：**
   ```bash
   git push origin main
   ```

### 重要提醒：
- 第3步中的 `your-actual-branch-name` 需要替換成你實際要合併的branch名稱
- 如果在合併過程中出現conflict，需要先解決衝突再繼續
- 建議先確認你的branch已經commit了所有變更

這個流程是標準的Git合併流程，適合將功能branch合併到main branch。

## 01. 比較兩個branch的差異
```bash
git diff main sandbox
```

## 02. 刪除branch
```bash
git branch -d 科目與支付方式歸類邏輯一次性互動方案
```

## 03. 強制push到github
```bash
git push -f origin main
```

## 04. 將一個branch合併到另一個branch（非main）
要將一個branch合併到另一個branch（非main），流程是這樣的：
基本合併流程
1.	切換到目標branch（你想要合併進去的branch）
git checkout target-branch-name
2.	確保目標branch是最新的
git pull origin target-branch-name
3.	合併來源branch
git merge source-branch-name
4.	推送合併結果
git push origin target-branch-name
實際範例
假設你想將 feature-branch 合併到 develop branch：
1.	git checkout develop
2.	git pull origin develop
3.	git merge feature-branch
4.	git push origin develop
重要提醒
•	目標branch = 你要合併進去的branch
•	來源branch = 你要被合併的branch
•	合併前確保兩個branch都已經commit了所有變更
•	如果有衝突需要先解決衝突再繼續
•	可以用 git status 隨時檢查狀態
與合併到main的差別
唯一差別就是把指令中的 main 替換成你的目標branch名稱，其他流程完全一樣。這樣可以在不同的開發branch之間進行合併，不一定要通過main branch。
