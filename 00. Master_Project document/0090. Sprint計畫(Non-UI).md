## I. 計畫概覽

### I.1 開發策略
1. 採用敏捷開發方式，以6個Phase漸進式交付，每個Phase專注於特定功能領域。
2. 在PL層分成UI與業務邏輯兩部分，本計畫專注於業務邏輯部分，亦即非UI的代碼開發。
3. 依照專案狀況彈性調整各Phase開發期程。
  
### I.2 技術架構
- **PL層（展示層）：** Flutter (APP) + LINE OA
- **APL層（應用邏輯層）：** 轉發API窗口
- **ASL層（應用服務層）：** 轉發API窗口
- **BL層（商業邏輯層）：** Node.js模組
- **DL層（資料層）：** Firebase Authentication + Firestore + Cloud Storage
