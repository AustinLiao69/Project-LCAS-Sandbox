## 1. 簡介
此文件用於描述使用者在系統/APP 中的操作流程，協助產品、設計、開發、QA 對齊需求。  
內容涵蓋 **流程圖、詳細步驟、對應 User Story、畫面及 API 映射**。  

---

## 2. 範圍 (Scope)
- 涵蓋範圍：登入、記帳、生成報表  
- 不涵蓋範圍：後台管理功能  

---

## 3. 前置條件 (Preconditions)
- 使用者已安裝 APP  
- 使用者已有有效帳號  

---

## 4. User Flow 圖表
```mermaid
flowchart TD
  A[開啟 App] --> B[登入]
  B -->|成功| C[首頁]
  B -->|失敗| D[顯示錯誤訊息]
  C --> E[新增記帳]
  E --> F[送出記帳資料]
  F --> G[顯示成功訊息]
5. 流程詳述 (Flow Details)
步驟編號	使用者操作	系統回應	備註
1	開啟 App	顯示登入頁	-
2	輸入帳號/密碼	系統驗證帳號密碼	若錯誤，跳至步驟 4
3	驗證成功	進入首頁	-
4	驗證失敗	顯示錯誤訊息	可重試
5	點擊「新增記帳」	顯示新增記帳表單	-
6	填寫金額與類別，送出	系統存入資料庫	-
7	建立成功	顯示「新增成功」訊息	-

6. 對應 User Story
US-001: 作為使用者，我希望能輸入帳號密碼登入

US-002: 作為使用者，我希望登入失敗時收到錯誤提示

US-010: 作為使用者，我希望能新增一筆記帳紀錄

7. 對應畫面 (Screen Mapping)
步驟	畫面 ID	畫面名稱
1	S-001	登入頁
3	S-002	首頁
4	S-003	錯誤訊息彈窗
5	S-004	新增記帳頁
7	S-005	成功提示頁

8. 對應 API (API Mapping)
步驟	API 名稱	Method	Endpoint	說明
2	驗證帳號密碼	POST	/api/auth/login	驗證使用者帳號
6	建立記帳紀錄	POST	/api/transactions	新增交易資料

9. 異常流程 (Alternative / Exception Flow)
異常 1：使用者輸入錯誤密碼 → 顯示錯誤訊息並允許重試

異常 2：伺服器無回應 → 顯示「系統繁忙」提示

10. 後置條件 (Postconditions)
成功：使用者已登入並完成記帳紀錄新增

失敗：使用者停留於登入頁或新增記帳頁，需再次操作
